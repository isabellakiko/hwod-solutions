## 什么是 ADR

ADR 是记录重要架构决策的文档，包括：
- 决策背景
- 考虑的方案
- 最终选择及理由
- 影响和后果

## ADR-001: 选择 React + Vite 作为前端技术栈

**日期**: 2025-12-01
**状态**: 已接受

**背景**: 需要选择前端技术栈构建算法题库展示系统

**决策**: 使用 React 19 + Vite 7

**理由**:
- React 生态成熟，组件化开发效率高
- Vite 构建速度快，开发体验好
- 团队熟悉 React

---

## ADR-002: 使用 JSON 文件存储题目数据

**日期**: 2025-12-01
**状态**: 已接受

**背景**: 需要选择题目数据存储方式

**决策**: 使用独立 JSON 文件存储每道题目

**理由**:
- 无需后端，部署简单
- 便于版本控制
- 支持动态导入，按需加载

---

## ADR-003: 部署到 GitHub Pages

**日期**: 2025-12-02
**状态**: 已接受

**背景**: 需要选择部署平台

**决策**: 使用 GitHub Pages

**理由**:
- 免费托管
- 与代码仓库集成
- 使用 gh-pages 包简化部署

3. 填写决策内容
4. 更新本索引文件
